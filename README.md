## 📋 About the Competition

**Meshtastic Build-Off 2026** is an open-source hackathon celebrating innovation in **LoRa mesh networking**. We're inviting developers, makers, and enthusiasts worldwide to build creative projects using the Meshtastic ecosystem.

Whether you're enhancing communication protocols, building practical applications, creating educational content, or pushing the boundaries of mesh networking—this is your stage.

**Timeline:** May – August 2026  
**Location:** Global (100% online)  
**Total Prize Pool:** $3,000+

---

## 🎯 What We're Looking For

We welcome projects across all categories:

- **Communication Apps** — Chat clients, messaging platforms, emergency systems
- **Hardware Hacks** — Custom devices, antennas, solar-powered nodes, gateways
- **Software Tools** — Network monitoring, analytics, deployment utilities, SDK extensions
- **Educational Content** — Tutorials, documentation, courses, open-source guides
- **Creative Uses** — Art projects, research applications, novel use cases

The only requirement: Your project must involve Meshtastic in a meaningful way.

## 📋 What We Need From You

When you submit, prepare:

| Field | Required | Details |
|-------|----------|---------|
| **Your Name** | Yes | Full name or team name |
| **Email** | Yes | For contact & winner notification |
| **Phone Number** | Yes | With country code (e.g., +1-555-123-4567) |
| **GitHub Repository Link** | Yes | Public repo with your project code |
| **Project Description** | Yes | Brief overview & key features (200+ characters) |
| **Tech Stack** | Yes | Languages, frameworks, libraries used |
| **Message to Organizers** | No | Anything special we should know |

**Tip:** Make sure your GitHub repository is **public** and includes a detailed README explaining:
- What your project does
- How to install & run it
- Key features & use cases
- Technologies & dependencies
- How it uses Meshtastic

## 💡 Tips for Success

1. **Start Early** — Building takes time; don't wait until August
2. **Document Everything** — Clear READMEs & comments help judges understand your work
3. **Show Your Process** — Commit history, progress updates, challenges overcome
4. **Test Thoroughly** — Ensure your project runs without errors
5. **Tell a Story** — Help judges see the vision & impact of your work
6. **Engage with Community** — Share updates, ask questions, collaborate with others

## ⚖️ Rules & Code of Conduct

- All submitted code must be **open-source** (MIT, Apache 2.0, GPL, or similar)
- You must have **rights** to all code & assets you submit
- Projects can be **solo or team-based** (no team size limit)
- Submissions must be your **original work** (reusing open-source libraries is fine)
- Be **respectful & inclusive** — read our [Code of Conduct](#)

Violations of these rules will result in disqualification.
